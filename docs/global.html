Details

Properties:
Name Type Description
current_tab string

declare the current tab of the details
loading boolean

declare the loading state of the details
show boolean

declare the show state of the details to show it or not
tabs Tabs

object contain all tabs information
type string

To define the type of details (optional)

  • PRIMARY => primary from the same screen when Add: add an empty row to the details
  • FOREIGN => from foreign screen when Add: open a modal from another screen
viewOnly boolean

some details just for only display purposes (mandatory)
recordDetailPropName string

the name of dtl property in the record usually is the same as the tab name
activeForeignList string | null
detailsRowIndex string | number | null

define the index of the row which has an action to easily track it
foreignKeys Array

list of foreign keys
pageURL Object

pageURL contain information used in getDetails in add mode

  • add mode => when save a details in a add mode we should display the added (saved)
  • we need to receive the details from the server after save add
  • to form a url to get the record details we need to know the following
  • master => knowing the name of the primary key of the master enable us to access its value form the record (we should know the value to send the request and receive the details of that record)
  • temp => is the url of getting record (note: record contain the details that we want)
addState boolean

to define the add state enabled or disabled: needed in limit record numb (optional)
recordsNum number

to define the max number of records allowed to be added used in limitAddHandlers (optional)
